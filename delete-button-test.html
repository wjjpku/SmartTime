批量删除按钮修复测试

问题描述：

修复前：
批量删除按钮可能显示重复文字，如"删除本日删除本日"
原因：customButtons配置中直接调用getDeleteButtonText()，useEffect中又通过DOM操作更新文本

修复方案：

修复后：
1. 将customButtons中deleteButton的text属性设置为空字符串
2. 完全依赖useEffect中的DOM操作来更新按钮文本
3. 添加延迟和重试机制确保DOM元素已渲染
4. 分别处理组件初始化和视图切换时的按钮文本更新

测试步骤：

  1. 打开智能任务管理系统主页
  2. 切换不同的日历视图（日视图、周视图、月视图）
  3. 检查右上角的删除按钮文本是否正确显示
  4. 验证按钮文本不会出现重复

预期结果：

日视图 (timeGridDay)
预期按钮文本: 删除本日
功能: 删除当前选中日期的所有任务
周视图 (timeGridWeek)
预期按钮文本: 删除本周
功能: 删除当前选中周的所有任务
月视图 (dayGridMonth)
预期按钮文本: 删除本月
功能: 删除当前选中月的所有任务

修复验证：

✅ 修复完成
• customButtons中deleteButton的text属性已设置为空字符串
• useEffect中添加了延迟和重试机制
• 分别处理了组件初始化和视图切换的按钮文本更新
• 按钮文本现在应该正确显示，不会出现重复
